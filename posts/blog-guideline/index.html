In the beginning, I supposed that I’m the only one who write-up thing in this blog. But now thing was change, this blog might have multiple bloggers. So it needs a guideline to describe how to contribute.

1. How to submit a new content

  • Here is the source repo. Fork it & start writing.
  • Create a pull request to submit your content.
  • Make sure to create your author page.

2. Create an author page

  • Create a directory under content/authors, name it as your desire nickname. For example, your name is amazingblogger.

3. Write a post

  • Very similar with author page, just place your post under content/posts.

    $ hugo new content/posts/a-new-post.md -t sam

  • A sample post.

    ---
title: "Blog Guideline"
date: 2019-08-22T14:40:59+07:00
comment: true
authors:
- kiennt
showDate: true
tags: ["blog", "tech"]
---

  • Multiple authors feature is supported.

  • You can disable or enable comment section with comment option. You might want to take look at how I create comment section.

  • Don’t forget to add a tag.

Just a script to init and update submodule, do git pull (both master and gh-pages branchs).

5.2. Publish to github page

I deploy the blog from gh-pages branch. You can also tell Github pages to treat your master branch as the published site or point to a separate gh-pages branch. The latter approach is a bit more complex but has some advantages:

  • It keeps your source and generated website in different branches and therefore maintains version control history for both.
  • Unlike the preceding docs/ option, it uses the default public folder.

The publish_to_ghpages.sh automates the set up steps.
